# ストアページUX改善計画

## 対応する問題
1. 「探す」が4種類混在 → タブUIで統合
2. 店舗検索が最下部で到達困難 → 独立ページ化 + ナビに導線
5. ページの役割が曖昧 → ストア（商品カタログ）と店舗検索を分離

## 現状のセクション構成（store/page.tsx）
```
1. ヒーロー（フラワーコレクション）
2. バリュープロポジション
---
3. 今季のおすすめ（横スクロール商品カード）
4. シーンで探す
5. スタイルで探す
---
6. ご予算から探す
7. マエストロのご紹介
8. 新着アレンジメント
9. すべてのアレンジメント
---
10. 店舗・アトリエを探す ← ShopFinder
```

## 改善後の構成

### A. `/store` — 商品カタログページ
```
1. ヒーロー（フラワーコレクション）
2. バリュープロポジション
---
3. 今季のおすすめ（横スクロール）
4. 探す（タブ切り替え）
   [シーン] [スタイル] [予算]  ← 3つを1セクションにタブ統合
---
5. マエストロのご紹介
6. 新着アレンジメント
7. すべてのアレンジメント
```
- 店舗セクション（ShopFinder）は削除
- 「シーンで探す」「スタイルで探す」「ご予算から探す」を1つのタブUIセクションに統合
- 各タブの中身は既存のUIをそのまま活用

### B. `/shops` — 店舗・アトリエ検索ページ（新規）
```
1. ヒーロー（店舗・アトリエを探す）
2. ShopFinder コンポーネント（既存のフィルタ付き検索UI）
```
- 既存の `ShopFinder` コンポーネントをそのまま使用
- 独自のヒーローセクションを追加してページとしての体裁を整える

### C. ナビゲーション更新
- TopNav / MobileMenu の navItems に「店舗」を追加
  ```ts
  { label: "ストア", href: "/store" },
  { label: "店舗", href: "/shops" },    // ← 追加
  { label: "注文する", href: "/order" },
  ```

## 対象ファイル
1. `src/app/store/page.tsx` — 探す系3セクションをタブUIに統合 + ShopFinder削除
2. `src/components/store/BrowseTabs.tsx` — 新規。タブ切り替えコンポーネント（Client Component）
3. `src/app/shops/page.tsx` — 新規。店舗検索の独立ページ
4. `src/components/layout/TopNav.tsx` — navItemsに「店舗」追加
5. `src/components/layout/MobileMenu.tsx` — navItemsに「店舗」追加

## タブUIの設計
```
┌─────────────────────────────────────────────┐
│  [シーンで探す]  [スタイルで探す]  [ご予算から探す]  │
├─────────────────────────────────────────────┤
│                                             │
│  （選択中のタブに対応するコンテンツ）              │
│                                             │
└─────────────────────────────────────────────┘
```
- タブはボーダー付きボタン、アクティブ状態はKintsugiゴールドの下線
- 各タブのコンテンツは既存のシーン/スタイル/予算UIをそのまま移植
- Client Component化（useState でタブ状態管理）

## 実装順序
1. `BrowseTabs.tsx` 作成（タブUI + 3つの探すコンテンツを内包）
2. `store/page.tsx` 更新（3セクション → BrowseTabs置き換え + ShopFinder削除）
3. `shops/page.tsx` 作成（独立ページ）
4. TopNav / MobileMenu 更新（「店舗」リンク追加）
5. ビルド確認 → Playwright確認
